# Customer Communication

## Overview

Support communication should be clear, respectful, and useful. Each update should help the customer understand the current state and whether action is needed from them.

## Communication Guidelines

### Use Plain Language

Avoid unnecessary technical detail in the first sentence. Add technical context when it helps explain the issue or next step.

### Be Specific

Name the affected service, device, site, or workflow when known.

### Avoid Overpromising

Do not promise fixed response times, guaranteed fixes, or confirmed root causes before they are known.

### Explain Next Steps

Customer updates should make clear whether BlackStorm is reviewing, waiting for information, planning follow-up, or closing the request.

## Suggested Update Structure

1. What changed
2. What it means
3. What happens next
4. Whether customer action is needed
