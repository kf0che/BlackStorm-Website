# Customer Portal Overview

The BlackStorm Customer Portal is planned as the customer-facing workspace for network monitoring visibility, support workflows, and account-level self-service.

The portal should feel calm, trustworthy, and operational. It should not look like a cluttered "NOC wall" or a high-pressure security console. Customers should be able to open the portal, understand what needs attention, and know what to do next.

## Portal Purpose

The portal should help customers answer simple operational questions:

- What is being monitored?
- Is anything currently down or degraded?
- What changed recently?
- Have alerts recovered?
- Who should receive notifications?
- Where can I ask for support?
- What reports or summaries are available?

## Trust Principles

The portal should:

- Use plain language
- Avoid panic-heavy colors or alerts
- Use clear status labels, not color alone
- Show placeholders only when features are not live
- Avoid compliance or certification claims
- Make customer data handling clear
- Give customers obvious support paths

## Accessibility Principles

The portal should support:

- Keyboard navigation
- Visible focus states
- Semantic headings
- Accessible table captions
- Status text in addition to status color
- Responsive layouts for phone and tablet use
- Form labels and descriptions for every input

## Future Scaling Goals

The portal structure should support future:

- RBAC
- Monitoring API integration
- Notification systems
- Customer self-service workflows
- Educational onboarding systems
- MSP-facing dashboards
- Mobile app adaptation
