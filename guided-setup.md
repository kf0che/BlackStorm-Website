# Guided Setup

Guided setup is the structured path customers follow to prepare their BlackStorm monitoring environment.

The guided setup should be short, plain-language, and forgiving. Customers should be able to pause, ask for help, and come back later.

## Setup Checklist

Recommended checklist items:

- Confirm account contact details
- Add business or organization name
- Add first customer site
- Identify primary network device
- Add additional monitored devices
- Choose alert contacts
- Review notification preferences
- Submit setup request

## Setup Wizard Principles

### One Topic Per Step

Each step should focus on one task. Avoid combining device inventory, contacts, and alert rules in a single screen.

### Explain Why

Before asking for technical information, explain why it is needed.

Example:

"We ask for the router or firewall first because it often tells us whether the entire site is reachable."

### Provide Examples

Use examples such as:

- Main office router
- Firewall
- Server
- Wireless access point
- Network switch

### Let Customers Skip Unknown Details

Some customers may not know IP addresses or device models. Let them provide a description and ask support to follow up.

## Tooltip and Help Strategy

Tooltips should explain small concepts without interrupting flow.

Contextual help panels should explain bigger concepts, such as:

- What counts as a monitored device
- How alerts work
- What an IP address is
- Who should receive notifications

## Mobile Usability Goals

Guided setup should work well on phones:

- Large tap targets
- Single-column forms
- Short input labels
- Step progress visible without crowding the screen
- Save or continue later option when available

## Future Scaling Goals

Guided setup should later support:

- API-driven device discovery
- Customer portal onboarding
- MSP-assisted onboarding
- Localization
- Troubleshooting walkthroughs
- Educational modules tied to each step
